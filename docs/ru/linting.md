# Линтинг (`portsmith check`)

Команда `portsmith check` проверяет пакеты в `internal/` на соответствие архитектурным правилам. Настройки задаются в **`portsmith.yaml`** в корне репозитория в секции `lint:`.

## Правило `call-pattern` (по желанию)

Проверяет именование **трёхуровневых вызовов** `receiver.field.method()` в файлах слоёв:

- **Handler:** файлы с именем `handler*.go`, кроме `handler_test.go`
- **Service:** только **`service.go`**

Это **не** файловый glob: в каждом паттерне ровно **три сегмента**, разделённые точкой. Сегмент `*` совпадает с любым одним Go-идентификатором.

Примеры:

| Паттерн | Смысл |
|--------|--------|
| `h.svc.*` | Ресивер только `h`, поле только `svc`, имя метода любое |
| `*.svc.*` | Любой ресивер, поле `svc`, имя метода любое |
| `*.service.*` | Любой ресивер, поле `service`, имя метода любое |

### Конфигурация

```yaml
lint:
  call_patterns:
    handler:
      allowed:
        - "*.svc.*"
      not_allowed:
        - "*.service.*"
    service:
      allowed:
        - "*.repo.*"
      not_allowed:
        - "*.repository.*"
```

- **`not_allowed`:** если вызов совпал с любым из паттернов, `portsmith check` выдаёт нарушение **`call-pattern`**.
- **`allowed`:** **не** включает строгий белый список (чтобы не ловить ложные срабатывания на посторонние трёхуровневые вызовы вроде `req.Header.Get`). Используется только как **подсказка** в тексте ошибки (`use "..." instead`).

Правило **выключено**, пока хотя бы у одного слоя не задан непустой список **`not_allowed`**. Список **`allowed`** влияет только на текст подсказки в сообщении и **сам по себе** правило не включает.

### Серьёзность (severity)

По умолчанию — **error**, когда `call_patterns` задан. Переопределение в `lint.rules`:

```yaml
lint:
  rules:
    call-pattern:
      severity: warning  # или off
```

### Ограничения

- Учитываются только **прямые** вызовы `recv.field.Method()`. Локальные алиасы вида `x := h.svc; x.Method()` не анализируются.
- Некорректные паттерны (не ровно три сегмента) при сопоставлении игнорируются.
