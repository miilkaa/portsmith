# TDD Guide

This guide shows how to apply Red → Green → Refactor in portsmith projects.

## The three-layer testing strategy

| Layer | Test type | Tools | Database |
|-------|-----------|-------|----------|
| Service | Unit | `mockery` mocks + `testing.T` | None |
| Handler | HTTP | `testkit.NewHTTPSuite` | None |
| Repository | Integration | `testkit.NewTestDB` (SQLite) | SQLite in-memory |

## Red → Green → Refactor

```
1. Red     — write contract tests that define the public API. They must fail.
2. Green   — write the minimal implementation to make tests pass.
3. Refactor — clean up the code. Tests must stay green.
4. Unit    — add edge cases, error paths, boundary conditions.
```

## Testing the service layer

```go
func TestOrderService_Create_success(t *testing.T) {
    // 1. Create a mock repository (generated by portsmith mock)
    repo := mocks.NewOrderRepository(t)
    repo.On("Create", mock.Anything, mock.AnythingOfType("*orders.Order")).Return(nil)

    // 2. Wire the service with the mock
    svc := orders.NewService(repo)

    // 3. Call and assert
    order, err := svc.Create(ctx, orders.CreateParams{Item: "book"})
    testkit.NoError(t, err)
    testkit.Equal(t, "book", order.Item)

    repo.AssertExpectations(t)
}
```

Key point: no database, no HTTP. The test is fast and isolated.

## Testing the handler layer

```go
func TestOrderHandler_Create(t *testing.T) {
    svc := mocks.NewOrderService(t)
    svc.On("Create", mock.Anything, mock.MatchedBy(func(p orders.CreateParams) bool {
        return p.Item == "book"
    })).Return(&orders.Order{ID: 1, Item: "book"}, nil)

    h := orders.NewHandler(svc)
    router := gin.New()
    h.Routes(router.Group("/api/v1"))

    suite := testkit.NewHTTPSuite(t, router)
    suite.POST("/api/v1/orders", `{"item":"book"}`).
        ExpectStatus(201).
        ExpectJSONPath("$.id", float64(1)).
        ExpectJSONPath("$.item", "book")
}
```

Key point: no database. Tests HTTP parsing and response serialisation.

## Testing the repository layer

```go
func TestOrderRepository_FindByID_notFound(t *testing.T) {
    db := testkit.NewTestDB(t, &orders.Order{})
    repo := orders.NewRepository(db.DB())

    _, err := repo.FindByID(ctx, 9999)
    if !apperrors.IsCode(err, apperrors.CodeNotFound) {
        t.Errorf("expected NOT_FOUND, got %v", err)
    }
}
```

Key point: SQLite in-memory — no Docker, fast startup. Tests actual SQL behaviour.

## Table-driven tests

```go
func TestOrderService_Create(t *testing.T) {
    testkit.Table(t, []testkit.Case{
        {
            Name: "success",
            Run: func(t *testing.T) {
                // ...
            },
        },
        {
            Name: "empty item returns error",
            Run: func(t *testing.T) {
                // ...
            },
        },
    })
}
```

## Running tests

```bash
# All tests
go test ./...

# With coverage
go test ./... -cover

# Specific package
go test ./internal/orders/...

# Verbose
go test ./... -v
```
